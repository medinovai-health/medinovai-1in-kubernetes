# Kubernetes HA Deployment — Tailscale Multi-Machine Guide

All machines are connected via **Tailscale**. Each machine runs a full K8s stack. One machine is designated as the DB primary (hosts postgres/redis). All others connect to it over the Tailscale tunnel.

## Install on Every Machine

### Machine A — Primary (hosts postgres + redis)

```bash
git clone git@github.com:myonsite-healthcare/medinovai-Deploy.git
cd medinovai-Deploy

make docker-up            # Start postgres, redis, prometheus, grafana
make k8s-install-primary  # Install K8s + configure as primary
```

At the end you'll see:
```
On OTHER machines:
  bash scripts/bootstrap/install-k8s.sh --db-host 100.79.214.33
```

---

### Machine B, C, D... — Secondary (connects to primary DB)

```bash
git clone git@github.com:myonsite-healthcare/medinovai-Deploy.git
cd medinovai-Deploy

make docker-up   # Start local prometheus + grafana (postgres/redis optional)
bash scripts/bootstrap/install-k8s.sh --db-host 100.79.214.33
```

Replace `100.79.214.33` with your primary machine's Tailscale IP.

## How It Works

| Layer | What | How |
|-------|------|-----|
| **Tailscale** | Encrypted mesh between all machines | Automatic, always-on |
| **Docker Compose** | Postgres, Redis, Prometheus, Grafana | Named volumes, `restart: unless-stopped` |
| **Kubernetes** | App services (6 pods) | K8s deployments, `traefik/whoami` stubs |
| **ConfigMap** | DB/Redis URL per machine | Auto-generated by `tailscale-config.sh` |
| **NodePort** | Cross-machine service access | `api-gateway:30080`, `auth-service:30081` |

K8s pods on Machine B reach postgres on Machine A at `100.79.214.33:5432` — through the Tailscale tunnel automatically.

## Networking Map

| Service | Local access | From other machines |
|---------|-------------|---------------------|
| api-gateway | `localhost:30080` or port-forward | `http://<ts-ip>:30080` |
| auth-service | `localhost:30081` or port-forward | `http://<ts-ip>:30081` |
| postgres | `localhost:5432` | `<primary-ts-ip>:5432` |
| redis | `localhost:6379` | `<primary-ts-ip>:6379` |
| Grafana | `http://localhost:3000` | `http://<ts-ip>:3000` |
| Prometheus | `http://localhost:9090` | `http://<ts-ip>:9090` |

## Commands Reference

```bash
# Setup (on each machine)
make tailscale-config                                    # Configure Tailscale (interactive)
make docker-up                                           # Start infra
make k8s-install                                         # Install K8s services

# Per-role shortcuts
make k8s-install-primary                                 # Primary (hosts DB)
bash scripts/bootstrap/install-k8s.sh --db-host 100.x.x.x  # Secondary

# Status
make k8s-status                                          # All pods
kubectl get svc -n medinovai-services                    # Services + NodePorts
tailscale status                                         # Tailscale peers

# Logs
kubectl logs -n medinovai-services -l app.kubernetes.io/name=api-gateway -f
kubectl logs -n medinovai-ai -l app.kubernetes.io/name=ai-inference -f

# Cross-machine smoke test
curl http://100.79.214.33:30080/health         # Hit Machine A's api-gateway from any machine
curl http://<machine-b-ts-ip>:30080/health     # Hit Machine B's api-gateway

# Uninstall / Reinstall
make k8s-uninstall                                       # Remove K8s only
make k8s-reinstall                                       # Clean reinstall
```

## Per-Machine Files (gitignored)

| File | Purpose |
|------|---------|
| `.env.tailscale` | This machine's Tailscale IP, DB host, credentials |
| `infra/kubernetes/overlays/docker-desktop/configmap-env-local.yaml` | K8s ConfigMap with Tailscale-resolved DB URLs |
| `infra/kubernetes/overlays/docker-desktop/configmap-env-ai-local.yaml` | Same for `medinovai-ai` namespace |

These are generated by `tailscale-config.sh` and never committed. Each machine gets its own.

## Firewall / Tailscale ACLs

Ports that must be open in your Tailscale ACL policy between machines:

| Port | Protocol | Direction | Purpose |
|------|----------|-----------|---------|
| 5432 | TCP | Primary only | Postgres |
| 6379 | TCP | Primary only | Redis |
| 30080 | TCP | All | api-gateway NodePort |
| 30081 | TCP | All | auth-service NodePort |
| 3000 | TCP | All | Grafana (optional) |
| 9090 | TCP | All | Prometheus (optional) |

Add to Tailscale ACL (`tailscale.com/admin/acls`):

```json
{
  "action": "accept",
  "src": ["tag:medinovai"],
  "dst": ["tag:medinovai:5432", "tag:medinovai:6379", "tag:medinovai:30080", "tag:medinovai:30081"]
}
```

Or simply allow all between your machines for testing (default Tailscale policy).

## Troubleshooting

| Problem | Fix |
|---------|-----|
| Pod can't reach DB | Verify `curl <ts-ip>:5432` from pod: `kubectl exec -n medinovai-services <pod> -- wget -q <ts-ip>:5432` |
| Tailscale IP changed | Re-run `make tailscale-config` then `make k8s-reinstall` |
| Wrong DB host | Check `.env.tailscale` — regenerate with `tailscale-config.sh --db-host <ip>` |
| NodePort unreachable | Ensure Docker Desktop is listening: `curl localhost:30080` — check Tailscale ACLs |
| context not set | `kubectl config use-context docker-desktop` |
